FLAG

这个电脑上的winrar不能强制打开破损的压缩文件。。。。。

数据包中的线索

打开是个数据包，用wireshark打开，观察发现，有一种类型的数据包里面的包含的数据特别多，追踪它的tcp流，发现了一大串疑似base64编码的东西，很长一串，因此排除直接转码得flag的情况，考虑将base64转图片，拿到flag。

假如给我三天光明

打开是一张图和一个压缩包，图片下面的一串点阵猜测是某种密码，解密出来是压缩包密码，google得知是一种盲文，成功打开压缩包，发现里面是个音频，听了一下猜测是摩斯密码，用剪辑软件打开，记录下摩斯密码再翻译出来：CTFWPEI08732?23DZ，发现还不是flag，试着改为小写，也不对，再把ctf删掉，成功！

ctf 3.16

二维码

打开是二维码，直接扫码后发现没有有用的信息，遂用010editor打开，发现

.png的文件尾是AE 42 60 82，按理来说在文件尾之后就应该结束，我们却发现png的文件尾后还跟着50 4B 03 04，这是.zip的文件头，说明在该照片下还隐藏着一个.zip文件，到这一步有两种方法：1.将50 4B 03 04之前的东西全部删掉，之后保存，将文件后缀名改为.zip 2.用Linux中的binwalk将隐藏的.zip文件分离出来。语句：binwalk -e 文件名

打开这个zip文件后，发现有加密，无法直接解压

先考虑伪加密，010editor打开，发现50 4B 01 02 后面是 09 00，说明是真加密

用archpr暴力破解密码即可

基础破解

发现是加密压缩包，archpr暴力破解后，txt文档里是一串base64编码，解码即可得到flag

文件中的秘密

打开属性，在备注中直接找到flag

LSB

在大多数PNG问图像中，没个像素都由R、G、B三原色组成，每种颜色一般用8位数据表示，如果修改其最低位，人眼是不能区分出这种微笑的变化的，因此可以利用没个像素的R、G、B颜色的分量的最低有效位来隐藏信息，这样每个像素可以携带3位的信息

根据题目提示，可以猜到大概率考的是这个知识点，用stegsolve打开，

直接看图片的最低有效位，发现隐藏了一个.png文件，将它保存下来，后缀名改为.png，打开发现是二维码，扫描即得flag

zip伪加密

根据题目，就知道考的是zip伪加密，010editor打开，将50 4B 01 02 后的 09 00 改为 00 00，就可直接解压得到txt文件内flag

git的使用

要使用命令行将文件提交到 GitHub 仓库中，你需要执行以下步骤：

  1. 克隆仓库（如果没有）： 如果你还没有将仓库克隆到本地，你需要先使用 git clone 命令将仓库克隆到本地。例如：

    git clone https://github.com/your-username/your-repository.git

  2. 进入仓库目录： 使用 cd 命令进入克隆下来的仓库目录：

    cd your-repository

  3. 添加文件到暂存区： 使用 git add 命令将要提交的文件添加到 Git 的暂存区。你可以添加单个文件或者多个文件。例如：

    git add file2.txt

    或者你也可以使用 git add . 命令将当前目录下所有文件添加到暂存区。

  4. 提交文件到本地仓库： 使用 git commit 命令将添加到暂存区的文件提交到本地仓库，并添加提交消息说明。例如：

    git commit -m "Add new files"

  5. 推送文件到远程仓库： 最后，使用 git push 命令将本地仓库的提交推送到 GitHub 远程仓库。你需要指定远程仓库的名称和分支。例如：

    git push origin main

    其中 origin 是远程仓库的名称，main 是要推送的分支名称。

执行完以上步骤后，你的文件就会被提交到 GitHub 仓库中了。
