import math
import random
 
class TSP:
    def __init__(self, num_cities, distance_matrix):
        self.num_cities = num_cities
        self.distance_matrix = distance_matrix
 
    def path_length(self, path):
        length = 0
        for i in range(len(path)-1):
            length += self.distance_matrix[path[i]][path[i+1]]
        length += self.distance_matrix[path[-1]][path[0]]
        return length
 
    def random_solution(self):
        path = list(range(self.num_cities))
        random.shuffle(path)
        return path
 
class SimulatedAnnealing:
    def __init__(self, tsp, T, Tmin, alpha):
        self.tsp = tsp
        self.T = T  
        self.Tmin = Tmin  
        self.alpha = alpha  
 
    def acceptance_probability(self, old_cost, new_cost, T):
        if new_cost < old_cost:
            return 1
        else:
            return math.exp(-(new_cost - old_cost) / T)
 
    def solve(self):
        current_solution = self.tsp.random_solution()
        current_cost = self.tsp.path_length(current_solution)
        while self.T > self.Tmin:
            new_solution = self.tsp.random_solution()
            new_cost = self.tsp.path_length(new_solution)
            if self.acceptance_probability(current_cost, new_cost, self.T) > random.random():
                current_solution = new_solution
                current_cost = new_cost
            self.T *= self.alpha
        return current_solution, current_cost

计组笔记

计算机结构及各部件功能

冯.诺依曼计算机

  1. 由运算器,存储器,控制器,输入设备,输出设备构成
  2. 指令和数据以同等地位存放于存储区内,可按地址寻访
  3. 指令和数据均用**二进制数表示
  4. 指令由操作码和地址码组成,操作码表示操作的性质,地址码表示操作数在存储器中的位置
  5. 指令在存储器内按顺序存放,通常,指令是顺序执行的
  6. 机器以运算器为中心,但现代计算机已转化为**以存储器为中心

部件功能介绍

  1. 运算器: 用来完成算术运算 和 逻辑运算 ,并将运算的中间结果暂存在运算器中
  2. 存储器 : 用来存放数据和程序
  3. 控制器 : 用来控制, 指挥程序和数据的输入,运行以及处理运算结果
  4. 输入设备 : 将人们熟悉的信息转化为机器能识别的形式
  5. 输出设备 : 将机器的运算结果转化为人们熟悉的信息形式

主存储器

  1. 指令和数据按地址存取
  2. 存储体 -> 存储单元 -> 存储元件, 存储元件就相当于一个bit, 一个存储单元存储n个bit, 称这串二进制代码为一个存储字, 它的位数称为存储字长(一个存储字还可以代表一条指令)
  3. **注意区分字和字节的区别! : 字是储存单元, 字节是固定的8位二进制代码,一字节 = 1B
  4. 每个存储单元都有一个编号, 称为存储单元的地址号

寄存器

  1. MAR(存储器地址寄存器) : 存放欲访问的存储单元的地址, 其位数对应存储单元的个数
  2. MDR(存储器寄存器) : 存放从存储体某单元取出的代码或者准备往某存储单元存入的代码, 其位数与存储字长相等
  3. ACC(累加器) : 存储被加数以及和, 被减数以及差, 乘积高位, 被除数以及余数
  4. MQ(乘商寄存器) : 存储乘数以及乘积低位, 商
  5. X(操作数寄存器) : 加数, 减数, 被乘数, 除数
  6. PC(程序计数器) : 存放当前欲执行指令的地址, 与MAR之间有通路且可以自动加一, 故能自动形成下一条指令的地址
  7. IR(指令寄存器) : 存放当前的指令, 来自于MDR

技术指标

  1. 机器字长 : CPU一次能处理数据的位数
  2. 存储容量 : a) 主存容量 : 主存中存放二进制代码的总位数: 存储单元数 * 存储字长 = MAR * MDR b) 辅存容量 : 字节数表示
  3. 运算速度 : a) MIPS : 单位时间内执行n个一百万条指令 b) CPI : 执行一条指令所需的时钟周期数 c) FLOPS : 每秒进行浮点运算的次数
    主存的作用: 负责临时存储正在使用的数据和程序，提供快速访问。其作用是为CPU提供运行所需的数据和指令，速度快，但断电后数据会丢失。
    辅存的作用 : 负责长期存储数据和程序，提供大容量存储。其作用是保存不常用但需要长期存储的数据，速度相对较慢，但数据断电后不会丢失。

总线结构

单总线

CPU, 主存 I/O设备都挂在同一总线上
优点 : 结构简单, 便于扩充
缺点 : 不允许两个以上的部件在同同一时刻向总线传输信息, 控制信号的传播存在严重延迟, 影响系统工作效率

双总线

将速度较低的I/O设备从单总线上分离, 形成主存总线与I/O总线分开的结构.(两个总线通过通道连接)
优点 : CPU下放部分功能给通道, 系统吞吐能力更大, 提高了操作间的并行性, 缩短了指令周期
缺点 : 增加了硬件复杂度和成本

三总线

  1. 将速率不同的I/O设备分类,接在不同通道上,主存总线用于CPU与主存之间的传输; I/O总线供CPU与各类I/O设备之间传递信息; DMA总线用于高速I/O设备与主存之间直接交换信息.(任一时刻只能只用一种总线)
  2. CPU与Cache由局部总线连接, 主存和Cache由系统总线连接, 扩展总线可接入很多接口, 支持I/O设备, 同时通过拓展总线接口与系统总线相连.
    优点 : 提升效率

四总线

与三总线类似, 但又多了一条高速总线挂接高速I/O设备.

**GPU: graphic processing unit, 图形处理器

总线判优控制

用以判断哪个主设备有用总线的优先使用权(主设备对总线有控制权, 从设备只能响应主设备发来的总线命令)

集中式

将控制逻辑集中在一处(例: CPU)

  1. 链式查询: 离总线控制部件最近的设备具有最高优先级. 特点: 只需很少几根线就能按一定优先次序实现总线控制, 并且很容易扩充设备, 但对电路故障很敏感, 且优先级别低的设备可能很难获得请求
  2. 计数器定时查询: 当某个请求占用总线的设备地址与计数值一致时, 便获得总线使用权. 计数器的初始值可以修改, 因此优先次序可以修改. 特点 : 对电路故障不敏感, 但增加了控制线数, 控制复杂
  3. 独立请求方式: 每台设备单独有一对总线请求线和总线同意线. 总线控制部件通过排队电路, 决定响应哪台设备的请求. 特点 : 响应速度快, 优先次序灵活(通过程序改变) , 但控制线数量多, 总线控制更复杂

分布式

将控制逻辑分散在与总线连接的各个部件或设备上

总线通信控制

主要解决通信双方如何获知传输开始和传输结束, 以及通信双方如何协调如何配合.

同步通信

通信双方由统一时标(通常是CPU的总线控制部件发出)控制数据传送
优点 : 规定明确统一, 模块间的配合简单一致
缺点 : 必须按最慢速度的部件来设计公共时钟, 影响工作效率, 给设计带来局限性, 缺乏灵活性
一般用于总线长度短, 各部件存取时间比较一致的场合

异步通信

采用应答方式(握手方式), 主模块发出请求信号,等从模块反馈回来”响应” 信号后才开始通信(主从模块之间要增加两条应答线)
应答方式又分三种类型

  1. 不互锁方式: 主模块发出请求信号后,不必等待接到从模块的回答信号, 而是经过一段时间, 确认从模块已收到请求信号, 便撤销其请求信号; 从模块同理
  2. 半互锁方式 : 主模块发出请求信号后, 必须待接到从模块的回答信号后再撤销请求信号; 而从模块不必
  3. 全互锁方式 : 主从模块都必须在接收到对方的回答后再撤销信号

半同步通信

所有地址, 命令, 数据信号的发出时间, 都严格参照系统时钟的某个前沿开始, 接收方都采用系统时钟后沿时刻来进行判断识别
通过插入时钟(等待)周期的措施来协调通信双方的配合问题

分离式通信

在准备数据的过程中, 总线可以接受其他模块的请求, 保证总线在被占用时都在做有效工作

存储器分类

随机存储器RAM

可读可写存, 存储器的任何一个存储单元的内容都可以随机存取, 却存取时间与存储单元的物理位置无关. 计算机系统的主存都采用RAM
静态 : SRAM(触发器原理寄存信息)
动态 : DRAM(电容充放电原理寄存信息)

只读存储器ROM

一旦存入原始信息, 就只能进行读取操作, 不能更改信息
用于存放固定不变的程序, 常数, 汉字字库…

输入输出系统

I/O指令

命令码 : 体现I/O设备的具体操作
设备码 : 多台I/O设备的选择码, 相当于设备的地址

I/O设备与主机的联系

I/O设备与主机采用总线连接的优势 :

  1. 数据经过接口既起到缓冲作用, 又可完成串-并的变换
  2. 接口能满足中断请求处理的要求, 使I/O设备与CPU可按并行方式工作, 提高效率
  3. 多台I/O设备分时占用总线, 使多台I/O设备互相之间也可以实现并行工作方式

I/O设备编址方式:

  1. 统一编址 : 将I/O地址看作存储器地址的一部分,(在存储器的存储空间中专门划出一段范围的地址作为I/O设备的地址) 所用访问指令与访存指令相似
  2. 不统一编址 : I/O地址和存储器地址分开, 访问需要专用的I/O指令

I/O设备与主机信息传送的控制方式

程序查询方式, 程序中断方式, 直接存储器存取方式(DMA), I/O通道方式, I/O处理机方式

  1. 程序查询方式 : 由CPU通过程序不断查询I/O设备是否已做好准备, 从而控制I/O设备与主机交换信息(要求I/O接口内设置一个能反映I/O谁被是否准备就绪的状态标记, CPU通过它得知I/O设备的准备情况) 缺点: CPU需要不断查询; 串行工作 -> 效率低
  2. 程序中断方式 : 当I/O设备准备就绪并向CPU发出中断请求后, CPU才中断现行程序, 转至中断服务程序
  3. DMA方式 : I/O设备通过DMA数据通路直接和主存进行信息交换, 跳过CPU, 不需要调用中断服务程序.(若DMA和CPU同时访问主存, DMA有优先权(窃取/挪用)占一个存取周期, 这期间CPU仍能进行内部操作)

计算机的运算方法

原, 补, 反, 移

  1. 符号位放在最前面, 0正, 1负
  2. 原码 : 符号+绝对值
  3. 补码 : 正数 : 和原码相同; 负数 : 原码取反(符号位除外)加1 补码 -> 原码: 反过来即可
  4. 反码 : 正数 : 和原码相同; 负数 : 原码取反(符号位除外)

数据寻址

![[19c4ee359e160b1a3f06b419e454347.png]]
真实地址 EA, 由寻址方式(寻址特征)和形式地址共同确定

  1. 立即寻址 : “#”立即寻址特征, A存储操作数本身(数据: 补码) 优点 : 可以直接获取操作数 缺点 : 范围限制
  2. 直接寻址 : EA = A 优点 : 寻找操作数比较简单 缺点: 寻址范围限制; 改A才能改EA
  3. 隐含寻址 : 指令字中不明显地给出操作数的地址, 其操作数的地址隐含在操作码或者某个寄存器中 优点: 缩短指令字长
  4. 间接寻址 : A指出操作数有效地址所在存储单元地址 ![[a60e8d52d2d6ce773b7ce3d242fa2f2.png]] 优点 : 寻址范围大; 便于编制程序(能很方便完成子程序返回) ![[fbae0638389600ec1cd74f5532fe8f8.png]] 缺点: 需要访存多次
  5. 寄存器寻址 : EA = 寄存器的编号 优点: 无需访问主存,节省时间; 指令字短, 节省存储空间![[26a5c228497c885bd4b59496622b8fc.png]]
  6. 寄存器间接寻址 : EA = 寄存器编号, 寄存器储存操作数所在主存单元的地址号(比间接寻址少访存一次)![[a4bb06900c4f5497258e6ef1e435d08.png]]
  7. 基址寻址 : 需设有基址寄存器BR, EA = 形式地址A + BR中的内容 隐式: 专门有一个BR, 只需指令的寻址特征位反映出基址寻址即可 显式 : 在一组通用寄存器里, 由用户明确指出哪个寄存器用作基址寄存器, 存放基地址 优点 : 扩大寻址范围
  8. 变址寻址 : EA = 形式地址A + 变址寄存器IX的内容![[2557be1f2f68a2a4a0e357abcf1050f.png]]
    基址和变址的区别 : 应用场合不同 : 基址用于位程序或者数据分配存储空间,因此BR内容不可变, A可变; 变址用于处理数组, A不可变, 而IX变
  9. 相对寻址 : EA = 程序计数器PC的内容 + A, 用于转移类指令, 转移后的目标地址与当前指令有一段距离, 为相对位移量由A给出(位移量)
  10. 堆栈寻址

指令格式设计

RISC

主要特征

计算机执行程序所需时间 = 机器指令数 × 每条机器指令所需的平均机器周期 × 每个机器周期的执行时间
特点:

  1. 选取频度较高的一些简单指令以及一些很有用但又不复杂的指令, 让复杂指令的功能由频度较高的简单指令的组合来实现
  2. 指令长度固定,指令格式种类少, 寻址方式种类少
  3. 只有存数/取数指令访问存储器, 其余指令操作在寄存器内完成
  4. CPU有多个通用寄存器
  5. 采用流水线技术, 大部分指令在一个时钟周期内完成
  6. 控制器采用组合逻辑控制, 不用微程序控制
  7. 采用优化的编译程序

机器周期

可看作所有指令执行过程中的一个基准时间, 机器周期取决于指令的功能以及器件的速度
通常以访问一次存储器的时间为基准时间作为机器周期

控制方式

控制单元控制一条指令执行的过程可看为依次执行一个确定的微操作序列的过程
CU的控制方式 : 如何形成控制不同位操作序列所采用的时序控制方式

  1. 同步控制方式 : 任何一条指令或指令中的任何一个微操作的执行都是事先确定的, 并且都受统一基准时标的时序信号所控制的方式(1. 定长的机器周期 2. 不定长的机器周期 3. 中央控制和局部控制相结合)
  2. 异步控制方式 : 不存在基准时标信号, 操作需要多长时间就占用多长时间, 但需要有应答线路控制
  3. 联合控制方式 : 异步+同步; 大部分统一对待,小部分区别对待
  4. 人工控制方式

微操作的节拍安排

  1. 有些微操作的次序不能改变
  2. 被控制对象不同的微操作, 若能在一个节拍内执行, 则尽可能安排在一个节拍内
  3. 有些微操作所占时间不长, 尽量安排在一个节拍内完成,并允许有先后次序

组合逻辑设计

优点 : 思路清晰, 简单明了
缺点: 线路结构庞杂, 调试困难

微程序设计

优点 : 不需要队逻辑表达式进行化简, 不用考虑逻辑门级数和门的扇入系数, 使设计简便; 控制信号 -> 二进制代码, 只要修改代码就可以改变操作内容, 利于计算机仿真

Rethinking Graph Backdoor Attacks: A Distribution-Preserving Perspective

Problem Definition

In-Distribution Constraint on Triggers

Our in-distribution constraint on trigger $g_t$ is defined as :
$$
f_o(g_i) < \tau$$
$f_o(g_i)$ is the anomaly score of $g_i$ and $\tau$ is a threshold which can be tuned based on datasets.

Distribution Preserving Graph Backdoor Attack

This objective si achieved by solving:
$$\min_{\theta_{g}}\sum_{v_i\in\nu_L}l(f_{\theta^*s}(\tilde{v}i),y_t)$$
$$\theta^*s=arg \min{\theta_s}\sum{v_i\in\nu_L}l(f_s(v_i),y_i)+\sum{v_i\in\nu_P}l(f_s(\tilde{v}_i),y_t)$$

Methodology

Chllenges

  1. how to generate ID triggers that have the capability to bypass outlier detection defense methods
  2. how to learn the trigger generator to obtain triggers that meet ID constraint while maintaining a high attack success rate.

In-Distribution Triggers Generation

key: the adversarial learning

Using an MLP as the adaptive trigger generator $f_g$, which generate triggers based on the target node’s node attributes.
$$h^m_i = MLP(x_i), \ \ X^g_i = W_fh^m_i, \ \ A^g_i = W_ah^m_i$$
$x_i$ is the node attributes of $v_i$, and $W_f$, $W_a$ are the learnable parameters for feature and structure generation.

An OOD $f_d$ refines its ability to discern between clean inputs and generated triggers by minimizing the binary classification loss. Concurrently, the generator $f_g$ craft triggers to deceive $f_d$ by maximizing the binary classification loss:
$$\min_{\theta_g}\max_{\theta_d} \mathcal{L}D=\sum{v_i\sim\mathcal{V}s}log(f_d(v_i))+\sum{g\in\tau_P}log(1-f_d(g))$$
To obtain $\mathcal{V}_s$, we pretrain an auto-encoder on the original graph G and select samples whose reconstruction losses are close to the mean loss

Enhancing ID Trigger Effectiveness

Enhancing Memorization of Triggers

Our objective is to ensure that these triggers can guide the surrogate classifier $f_i$ to learn a high cosine similarity between $z_s(\tilde{v}_i)$ and $v_s(\tilde{v}_j)$.
This approach ensures that the trigger attributes significantly impact the target node attributes.

Our goal is to generate triggers that guide the surrogate model to learn a high cosine similarity between enbeddings $z_s(\tilde{v}_i)$ and $z_s(v_j)$, while ensuring the similarity between $z_s(\tilde{v}_i)$ and $z_s(v_k)$ is lower. The loss function is:
$$\mathcal{L}E=-\sum{v_i\in\mathcal{V}U}\sum{v_j\in\mathcal{V}_U}S(z_s(\tilde{v}_i),z_s(\tilde{v}j))+\sum{v_i\in\mathcal{V}U}\sum{v_j\in\mathcal{V}_t}-log\left(\frac{S(z_s(\tilde{v}_i),z_s(v_j))}{S(z_s(\tilde{v}i),z_s(v_j))+\sum{v_k\in\mathcal{V}_L\backslash\mathcal{V}_t}S(z_s(\tilde{v}_i),z_s(v_k))}\right)$$
